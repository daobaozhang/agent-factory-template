# CREATE_NEW_PROJECT.md

## What This Is

This document explains how to create a new Agent Factory project instance from the v1.0 template, **without any chat context**.

---

## A. Step-by-Step: Create a New Project

### Step 1: Navigate to Projects Directory

```bash
cd /volume2/qa/projects
```

> This is where all Agent Factory instances live. Adjust the path if your environment differs.

### Step 2: Run create_project.sh

```bash
bash /path/to/template/scripts/create_project.sh my_new_project
```

Where `/path/to/template` is the template root directory (e.g. `/volume2/qa/projects/ppt_agent_factory`).

Example:

```bash
cd /volume2/qa/projects/ppt_agent_factory
bash scripts/create_project.sh my_new_project
```

This will:

1. Copy the template to `/volume2/qa/projects/my_new_project/`
2. Generate `config/template_version.json` with the current timestamp
3. Run `scripts/survey_environment.sh` → generates `config/ENV_PROFILE.json`
4. Initialize a git repository (if git is available)
5. Run `scripts/init_current_sprint.sh` → creates `docs/CURRENT_SPRINT.json`, `docs/CURRENT_SPRINT.md`, `runtime/status/system_ready.json`
6. Call `scripts/entrypoint.sh` → starts bootstrap based on `bootstrap.mode`

### Step 3: (Optional) Create Without Auto-Bootstrap

If you only want to create the project files without running the entrypoint:

```bash
bash scripts/create_project.sh my_new_project --no-bootstrap
```

### Step 4: (Optional) Force a Specific Bootstrap Mode

```bash
bash scripts/create_project.sh my_new_project --bootstrap-mode=manual
```

### Step 5: Navigate to Your New Project

```bash
cd /volume2/qa/projects/my_new_project
```

## B. What Files Are Generated

### Automatically Created During `create_project.sh`

| File | Generated By | Purpose |
|------|-------------|---------|
| `config/template_version.json` | create_project.sh | Template metadata with creation timestamp |
| `config/ENV_PROFILE.json` | survey_environment.sh | Runtime environment configuration |
| `docs/CURRENT_SPRINT.json` | init_current_sprint.sh | Machine-readable sprint state (SSOT) |
| `docs/CURRENT_SPRINT.md` | init_current_sprint.sh | Human-readable sprint state |
| `runtime/status/system_ready.json` | init_current_sprint.sh | Ready signal |
| `runtime/status/task_result.json` | init_current_sprint.sh | Latest task result pointer |
| `.git/` | create_project.sh (if git available) | Git repository |

### Copied From Template (Unchanged)

All other files are copied verbatim from the template:
- `skills/` — all role definitions and protocols
- `scripts/` — all execution scripts
- `services/` — ppt-gateway and orchestrator stub
- `runtime/schemas/` — task_result validation schema
- `runtime/scripts/` — validation script
- `config/ENV_PROFILE.template.json` — the template for ENV_PROFILE
- `config/template_version.json` — version metadata
- `docs/` — all protocol and context documents

## C. The Initialization Flow

### 1. survey_environment.sh

Detects the runtime environment automatically:

- Checks if `qwen` CLI is available → sets `executor`
- Checks if `docker` is available and daemon is running
- Detects if running on NAS (`/volume2` exists) → sets `environment`
- Detects gateway port (checks if 18080 is in use, defaults to 18081)
- Detects OpenClaw container name via `docker ps`
- Auto-selects `bootstrap.mode`: `qwen` if qwen is available, otherwise `manual`

**Output**: `config/ENV_PROFILE.json` — a file the rest of the system reads.

### 2. ENV_PROFILE.json

This file is **never** in the template. It is always generated at instantiation time.

Template contains only `config/ENV_PROFILE.template.json` with placeholders like `{{PROJECT_NAME}}`, `{{EXECUTOR}}`, etc.

### 3. init_current_sprint.sh

Creates the initial sprint state:
- `docs/CURRENT_SPRINT.json` — with `phase: "phase_0_bootstrap"`, `task_status: "pending"`
- `docs/CURRENT_SPRINT.md` — human-readable version
- `runtime/status/system_ready.json` — signals readiness

### 4. entrypoint.sh

Reads `config/ENV_PROFILE.json`, looks at `bootstrap.mode`, and branches:

| Mode | Behavior |
|------|----------|
| `qwen` | Calls `scripts/bootstrap_minimal_loop.sh` → runs the full dev → QA → validate → update CURRENT_SPRINT loop |
| `manual` | Prints a manual operation guide, takes no action |
| `openclaw_minimal` | Prints "waiting for PM scheduling", takes no action |
| Unknown | Exits with error code 1 |

### 5. bootstrap_minimal_loop.sh (the Closed Loop)

When called (qwen mode), executes this 9-step sequence:

1. **Infrastructure check** — verifies all required directories and files exist
2. **Generate task JSON** — creates `runtime/jobs/bootstrap_loop_001.json`
3. **Run dev** — calls `scripts/run_dev_task.sh` → produces `runtime/artifacts/bootstrap_loop_001_dev_result.json`
4. **Read dev result** — reads and validates the dev output
5. **Run QA** — calls `scripts/run_qa_task.sh` → produces `runtime/artifacts/bootstrap_loop_001_qa_result.json`
6. **Read QA result** — reads and validates the QA output
7. **Schema validation** — runs `runtime/scripts/validate_task_result.sh` against the schema
8. **Update CURRENT_SPRINT** — sets `last_task_id`, `last_task_status`, `last_result_path`, `last_updated`
9. **Output verdict** — prints `RESULT: PASS` or `RESULT: FAIL`

## D. Three Bootstrap Modes

### Mode: qwen

```bash
bash scripts/create_project.sh my_project --bootstrap-mode=qwen
```

**What it does**: Runs the full dev → QA → validate → CURRENT_SPRINT update loop automatically.

**Use when**: You have a working executor (qwen CLI or inline fallback) and want to verify the pipeline immediately.

**What happens**:
1. A bootstrap task is generated
2. Dev work is executed (creates an artifact)
3. QA verifies the dev work
4. Schema validation runs
5. CURRENT_SPRINT.json is updated with the result
6. You see `RESULT: PASS` or `RESULT: FAIL`

### Mode: manual

```bash
bash scripts/create_project.sh my_project --bootstrap-mode=manual
```

**What it does**: Creates the project, initializes state, prints instructions, and stops.

**Use when**: You want to inspect the project structure before running anything.

**What happens**:
- Project files are created
- ENV_PROFILE.json is generated
- Sprint state is initialized
- A manual operation guide is printed
- No tasks are executed

### Mode: openclaw_minimal

```bash
bash scripts/create_project.sh my_project --bootstrap-mode=openclaw_minimal
```

**What it does**: Waits for OpenClaw (PM) to schedule tasks.

**Use when**: OpenClaw is connected and will manage task orchestration.

**What happens**:
- Project files are created
- ENV_PROFILE.json is generated
- Sprint state is initialized
- The entrypoint prints "waiting for PM scheduling" and exits

## E. What Happens on First Run

When you run `create_project.sh` (default, no `--no-bootstrap`):

1. **Task is generated** → `runtime/jobs/bootstrap_loop_001.json`
2. **Dev executes** → creates `runtime/artifacts/bootstrap_loop_001_output.txt` and `runtime/artifacts/bootstrap_loop_001_dev_result.json`
3. **QA verifies** → creates `runtime/artifacts/bootstrap_loop_001_qa_result.json` with `status: "pass"` or `"fail"`
4. **CURRENT_SPRINT is updated** → `last_task_id`, `last_task_status`, `last_result_path` are set
5. **Latest pointer is written** → `runtime/status/task_result.json` is updated
6. **Result is printed** → `RESULT: PASS` or `RESULT: FAIL`

## F. Common Errors and Fixes

### ERROR: config/ENV_PROFILE.json not found

**Cause**: `survey_environment.sh` was not run, or `config/ENV_PROFILE.template.json` is missing.

**Fix**:
```bash
bash scripts/survey_environment.sh
```
Or manually copy the template:
```bash
cp config/ENV_PROFILE.template.json config/ENV_PROFILE.json
# Then edit config/ENV_PROFILE.json with correct values
```

### ERROR: docker not available

**Cause**: Docker is not installed or daemon is not running.

**Impact**: The system will still work for inline execution. Only OpenClaw container detection will fail (gracefully degrades to `none_detected`).

**Fix**: Install Docker or set `openclaw.container` manually in `config/ENV_PROFILE.json`.

### ERROR: qwen CLI not found

**Cause**: `qwen` is not in PATH.

**Impact**: `run_dev_task.sh` and `run_qa_task.sh` fall back to inline execution automatically. No failure — just a notice.

**Fix**: None required. Inline execution works without qwen.

### ERROR: Permission denied on scripts

**Cause**: Scripts are not executable.

**Fix**:
```bash
chmod +x scripts/*.sh runtime/scripts/*.sh
```

### ERROR: Unknown bootstrap.mode

**Cause**: `config/ENV_PROFILE.json` has an invalid `bootstrap.mode` value.

**Fix**:
```bash
# Edit the file and set to one of: qwen, manual, openclaw_minimal
python3 -c "
import json
cfg = json.load(open('config/ENV_PROFILE.json'))
cfg['bootstrap.mode'] = 'manual'
json.dump(cfg, open('config/ENV_PROFILE.json', 'w'), indent=2)
"
```

### ERROR: Port already in use

**Cause**: The default gateway port (18080 or 18081) is occupied.

**Impact**: Non-blocking. The system detects port conflicts and records them in `blockers` in CURRENT_SPRINT.json.

**Fix**: Update `gateway.port` in `config/ENV_PROFILE.json` to an available port.

### ERROR: Git init failed

**Cause**: Git is not installed, or directory is already a git repo.

**Impact**: Non-blocking. The project works without git.

**Fix**: Install git or run `git init` manually.
